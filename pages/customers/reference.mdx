## <Icon icon="folder-tree" type="solid" /> Estrutura

Um cliente é representado em nossa API pela seguinte estrutura:

```json
{
  "id": "clx1a2b3c4d5e6f7g8h9",
  "name": "João Silva",
  "email": "joao@email.com",
  "phone": "11987654321",
  "document": "12345678900",
  "documentType": "CPF",
  "address": {
    "street": "Rua Exemplo",
    "streetNumber": "123",
    "complement": "Apto 4",
    "neighborhood": "Centro",
    "zipCode": "01234567",
    "city": "São Paulo",
    "state": "SP",
    "country": "BR"
  },
  "createdAt": "2026-02-28T09:00:00.000Z",
  "updatedAt": "2026-02-28T09:00:00.000Z"
}
```

## Atributos:

<AccordionGroup>
  <Accordion title="id:">
    ```json
    { "id": "clx1a2b3c4d5e6f7g8h9" }
    ```
    `id` : <u>string.</u><br />
    Identificador único do cliente no formato CUID.
  </Accordion>

  <Accordion title="name:">
    ```json
    { "name": "João Silva" }
    ```
    `name` : <u>string.</u><br />
    Nome completo do cliente.
  </Accordion>

  <Accordion title="email:">
    ```json
    { "email": "joao@email.com" }
    ```
    `email` : <u>string.</u><br />
    Endereço de e-mail do cliente.
  </Accordion>

  <Accordion title="phone:">
    ```json
    { "phone": "11987654321" }
    ```
    `phone` : <u>string.</u><br />
    Telefone com DDD, somente números.
  </Accordion>

  <Accordion title="document:">
    ```json
    { "document": "12345678900" }
    ```
    `document` : <u>string.</u><br />
    CPF ou CNPJ do cliente, somente números. Usado como identificador único por projeto.
  </Accordion>

  <Accordion title="documentType:">
    ```json
    { "documentType": "CPF" }
    ```
    `documentType` : <u>string.</u><br />
    Tipo do documento. Pode ser `CPF` ou `CNPJ`.
  </Accordion>

  <Accordion title="address:">
    ```json
    {
      "address": {
        "street": "Rua Exemplo",
        "streetNumber": "123",
        "complement": "Apto 4",
        "neighborhood": "Centro",
        "zipCode": "01234567",
        "city": "São Paulo",
        "state": "SP",
        "country": "BR"
      }
    }
    ```
    `address` : <u>object.</u><br />
    Endereço do cliente. Todos os campos são opcionais.
  </Accordion>

  <Accordion title="createdAt:">
    ```json
    { "createdAt": "2026-02-28T09:00:00.000Z" }
    ```
    `createdAt` : <u>date-time.</u><br />
    Data e hora de criação do registro.
  </Accordion>

  <Accordion title="updatedAt:">
    ```json
    { "updatedAt": "2026-02-28T09:00:00.000Z" }
    ```
    `updatedAt` : <u>date-time.</u><br />
    Data e hora da última atualização.
  </Accordion>
</AccordionGroup>

<Note>
  O campo `document` é o identificador único do cliente por projeto. Se você criar uma transação com um `document` já cadastrado, o registro é atualizado automaticamente com os novos dados.
</Note>
