# Buscar cliente por ID

Retorna os dados completos de um cliente específico pelo ID gerado pela Bob Payments.

## Endpoint

```
GET /api/customers/:id
```

## Parâmetros de rota

<AccordionGroup>
  <Accordion title="id" icon="fingerprint">
    **string** · Obrigatório

    ID do cliente no formato CUID. Exemplo: `clx1a2b3c4d5e6f7g8h9`.
  </Accordion>
</AccordionGroup>

## Exemplo de requisição

<CodeGroup>
```bash cURL
curl https://api.payments.bob.company/api/customers/clx1a2b3c4d5e6f7g8h9 \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const customerId = 'clx1a2b3c4d5e6f7g8h9';

const response = await fetch(
  `https://api.payments.bob.company/api/customers/${customerId}`,
  {
    headers: { 'Authorization': 'Bearer sk_test_sua_chave' }
  }
);

const { data } = await response.json();
console.log(data.name); // "João Silva"
```

```python Python
import requests

customer_id = 'clx1a2b3c4d5e6f7g8h9'

response = requests.get(
    f'https://api.payments.bob.company/api/customers/{customer_id}',
    headers={'Authorization': 'Bearer sk_test_sua_chave'}
)

data = response.json()['data']
print(data['name'])
```
</CodeGroup>

## Erros comuns

| HTTP | Descrição |
|------|-----------|
| `404` | Cliente não encontrado ou não pertence ao seu projeto |
| `401` | Token de autenticação inválido ou ausente |

<Tip>
  Se você tem o CPF/CNPJ, e-mail ou telefone do cliente mas não o ID, use o endpoint de [busca por documento/email/phone](/pages/customers/lookup).
</Tip>
