# Checar status da transação

Retorna os dados completos de uma transação específica, incluindo o status atual de pagamento.

## Endpoint

```
GET /api/transactions/:id
```

## Parâmetros de rota

<AccordionGroup>
  <Accordion title="id" icon="fingerprint">
    **string** · Obrigatório

    ID da transação retornado ao criar a cobrança. Formato CUID: `clx7a8b9c0d1e2f3g4h5`.
  </Accordion>
</AccordionGroup>

## Exemplo de requisição

<CodeGroup>
```bash cURL
curl https://api.payments.bob.company/api/transactions/clx7a8b9c0d1e2f3g4h5 \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const transactionId = 'clx7a8b9c0d1e2f3g4h5';

const response = await fetch(
  `https://api.payments.bob.company/api/transactions/${transactionId}`,
  {
    headers: { 'Authorization': 'Bearer sk_test_sua_chave' }
  }
);

const { data } = await response.json();
console.log(data.status); // 'waiting_payment' | 'paid' | 'expired' | ...
```

```python Python
import requests

transaction_id = 'clx7a8b9c0d1e2f3g4h5'

response = requests.get(
    f'https://api.payments.bob.company/api/transactions/{transaction_id}',
    headers={'Authorization': 'Bearer sk_test_sua_chave'}
)

data = response.json()['data']
print(data['status'])
```
</CodeGroup>

<Note>
  O `pixCode` e `pixQrCode` ficam disponíveis apenas enquanto o status for `waiting_payment`. Após o pagamento ou expiração, esses campos retornam `null`.
</Note>

<Tip>
  Prefira usar [webhooks](/pages/webhooks) em vez de polling para receber notificações de pagamento em tempo real, sem precisar consultar o status repetidamente.
</Tip>

## Erros comuns

| HTTP | Descrição |
|------|-----------|
| `404` | Transação não encontrada ou não pertence ao seu projeto |
| `401` | Token de autenticação inválido ou ausente |
