# Simular pagamento (sandbox)

No ambiente sandbox, você pode simular a confirmação de um pagamento PIX sem movimentar dinheiro real. Após a simulação, o webhook configurado é disparado automaticamente.

<Warning>
  Este endpoint só funciona com `sk_test_*`. Tentativas com `sk_live_*` retornam erro.
</Warning>

## Endpoint

```
POST /api/sandbox/transactions/:id/pay
```

## Parâmetros de rota

<AccordionGroup>
  <Accordion title="id" icon="fingerprint">
    **string** · Obrigatório

    ID da transação sandbox a ser confirmada. A transação deve estar no status `waiting_payment`.
  </Accordion>
</AccordionGroup>

## Exemplo de requisição

<CodeGroup>
```bash cURL
curl -X POST https://api.payments.bob.company/api/sandbox/transactions/clx7a8b9c0d1e2f3g4h5/pay \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const transactionId = 'clx7a8b9c0d1e2f3g4h5';

const response = await fetch(
  `https://api.payments.bob.company/api/sandbox/transactions/${transactionId}/pay`,
  {
    method: 'POST',
    headers: { 'Authorization': 'Bearer sk_test_sua_chave' }
  }
);

const result = await response.json();
console.log(result.success); // true
```

```python Python
import requests

transaction_id = 'clx7a8b9c0d1e2f3g4h5'

response = requests.post(
    f'https://api.payments.bob.company/api/sandbox/transactions/{transaction_id}/pay',
    headers={'Authorization': 'Bearer sk_test_sua_chave'}
)

print(response.json())
```
</CodeGroup>

## Resposta de sucesso

```json
{
  "success": true,
  "message": "Transaction paid successfully",
  "data": {
    "id": "clx7a8b9c0d1e2f3g4h5",
    "status": "paid",
    "updatedAt": "2026-02-27T09:05:32.000Z"
  }
}
```

## O que acontece após a simulação

<Steps>
  <Step title="Status atualizado">
    A transação muda de `waiting_payment` para `paid` imediatamente.
  </Step>
  <Step title="Webhook disparado">
    Se você configurou uma URL de webhook no projeto, ela recebe um `POST` com o evento `transaction.paid`.
  </Step>
  <Step title="Fluxo completo testado">
    Você pode validar toda a sua lógica de negócio — liberar acesso, enviar e-mail, atualizar banco de dados — sem processar pagamentos reais.
  </Step>
</Steps>

<Tip>
  Configure um webhook antes de simular para testar o fluxo completo de ponta a ponta. Consulte a [documentação de webhooks](/pages/webhooks) para configurar.
</Tip>

## Erros comuns

| HTTP | Descrição |
|------|-----------|
| `400` | Transação não está em `waiting_payment` |
| `403` | Tentativa de usar com chave de produção (`sk_live_*`) |
| `404` | Transação não encontrada |
