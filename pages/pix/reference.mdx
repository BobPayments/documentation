## <Icon icon="folder-tree" type="solid" /> Estrutura

Uma transação PIX é representada em nossa API pela seguinte estrutura:

```json
{
  "id": "clx7a8b9c0d1e2f3g4h5",
  "externalId": "pedido_123",
  "amountCents": 10000,
  "product": "Plano Premium",
  "status": "waiting_payment",
  "pixCode": "00020126580014br.gov.bcb.pix...",
  "expirationDate": "2026-02-28T10:00:00.000Z",
  "customer": {
    "name": "João Silva",
    "email": "joao@email.com",
    "document": "12345678900"
  },
  "createdAt": "2026-02-28T09:00:00.000Z",
  "updatedAt": "2026-02-28T09:00:00.000Z"
}
```

## Atributos:

<AccordionGroup>
  <Accordion title="id:">
    ```json
    { "id": "clx7a8b9c0d1e2f3g4h5" }
    ```
    `id` : <u>string.</u><br />
    Identificador único da transação no formato CUID.
  </Accordion>

  <Accordion title="externalId:">
    ```json
    { "externalId": "pedido_123" }
    ```
    `externalId` : <u>string.</u><br />
    Identificador da transação no seu sistema. Use para cruzar dados sem armazenar o `id` interno. Deve ser único por projeto.
  </Accordion>

  <Accordion title="amountCents:">
    ```json
    { "amountCents": 10000 }
    ```
    `amountCents` : <u>integer.</u><br />
    Valor da transação em centavos. Exemplo: `10000` = R$ 100,00.
  </Accordion>

  <Accordion title="product:">
    ```json
    { "product": "Plano Premium" }
    ```
    `product` : <u>string.</u><br />
    Nome ou descrição do produto/serviço cobrado. Aparece na notificação do PIX para o pagador.
  </Accordion>

  <Accordion title="status:">
    ```json
    { "status": "waiting_payment" }
    ```
    `status` : <u>string.</u><br />
    Status atual da transação.
    <Info>
      | Status | Descrição |
      |--------|-----------|
      | `waiting_payment` | **Aguardando pagamento do cliente** |
      | `processing` | **Pagamento recebido, aguardando confirmação** |
      | `paid` | **Pagamento confirmado** |
      | `expired` | **O tempo limite de pagamento foi excedido** |
      | `cancelled` | **A transação foi cancelada** |
      | `refunded` | **O valor foi devolvido ao cliente** |
    </Info>
  </Accordion>

  <Accordion title="pixCode:">
    ```json
    { "pixCode": "00020126580014br.gov.bcb.pix..." }
    ```
    `pixCode` : <u>string.</u><br />
    Código PIX copia-e-cola (Pix Payload Format Object). Disponível apenas quando `status` é `waiting_payment`.
  </Accordion>

  <Accordion title="expirationDate:">
    ```json
    { "expirationDate": "2026-02-28T10:00:00.000Z" }
    ```
    `expirationDate` : <u>date-time.</u><br />
    Data e hora de expiração da transação. Após esse momento, o status muda para `expired`.
  </Accordion>

  <Accordion title="customer:">
    ```json
    {
      "customer": {
        "name": "João Silva",
        "email": "joao@email.com",
        "document": "12345678900"
      }
    }
    ```
    `customer` : <u>object.</u><br />
    Dados do comprador vinculado à transação. Consulte a [entidade Customer](/pages/customers/reference).
  </Accordion>

  <Accordion title="createdAt:">
    ```json
    { "createdAt": "2026-02-28T09:00:00.000Z" }
    ```
    `createdAt` : <u>date-time.</u><br />
    Data e hora de criação da transação.
  </Accordion>

  <Accordion title="updatedAt:">
    ```json
    { "updatedAt": "2026-02-28T09:05:32.000Z" }
    ```
    `updatedAt` : <u>date-time.</u><br />
    Data e hora da última atualização.
  </Accordion>
</AccordionGroup>
