# Adicionar Order Bump

Adiciona um produto adicional que será exibido no checkout para incentivar a compra junto. Cada link suporta até **5 order bumps**.

## Path Parameters

<ParamField path="id" type="string" required>
  ID do link de pagamento.
</ParamField>

## Request Body

<ParamField body="productId" type="string" required>
  ID do produto a ser adicionado como order bump.
</ParamField>

<ParamField body="customAmountCents" type="integer">
  Valor customizado em centavos (mínimo 100). Se omitido, usa o preço do produto.
</ParamField>

<ParamField body="heading" type="string">
  Título exibido acima do order bump (máx. 200 caracteres). Ex: `"Oferta especial!"`.
</ParamField>

<ParamField body="bumpDescription" type="string">
  Descrição do benefício do order bump (máx. 500 caracteres).
</ParamField>

<ParamField body="position" type="integer">
  Posição de exibição (baseada em 0). Se omitida, o bump é adicionado ao final.
</ParamField>

## Response (201)

```json
{
  "orderBump": {
    "id": "bump_01",
    "paymentLinkId": "clx7a8b9c0d1e2f3g4h5",
    "productId": "prod_abc",
    "position": 0,
    "active": true,
    "customAmountCents": null,
    "heading": "Adicione suporte prioritário",
    "bumpDescription": "Atendimento em menos de 2 horas úteis.",
    "createdAt": "2026-01-17T10:00:00.000Z"
  }
}
```

<ResponseField name="orderBump.id" type="string">
  ID único do order bump criado.
</ResponseField>
<ResponseField name="orderBump.position" type="integer">
  Posição de exibição no checkout (0 = primeiro).
</ResponseField>

## Erros

| HTTP | Código | Descrição |
|------|--------|-----------|
| 400 | `ERR_VALIDATION_001` | Body inválido |
| 401 | `ERR_AUTH_009` | Token inválido |
| 404 | — | Link ou produto não encontrado |
| 409 | — | Produto já adicionado ou limite de 5 bumps atingido |
| 500 | — | Erro interno do servidor |
