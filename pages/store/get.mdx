# Dados da loja

Retorna as informações e configurações do projeto associado à chave de API utilizada na requisição.

## Endpoint

```
GET /api/store
```

## Exemplo de requisição

<CodeGroup>
```bash cURL
curl https://api.payments.bob.company/api/store \
  -H "Authorization: Bearer sk_test_sua_chave"
```

```javascript JavaScript
const response = await fetch('https://api.payments.bob.company/api/store', {
  headers: { 'Authorization': 'Bearer sk_test_sua_chave' }
});

const { data } = await response.json();
console.log(data.name);       // "Minha Loja"
console.log(data.webhookUrl); // URL do webhook configurado
console.log(data.domains);    // domínios autorizados
```

```python Python
import requests

response = requests.get(
    'https://api.payments.bob.company/api/store',
    headers={'Authorization': 'Bearer sk_test_sua_chave'}
)

data = response.json()['data']
print(data['name'])
print(data['webhookUrl'])
print(data['domains'])
```
</CodeGroup>

```json
{
  "success": true,
  "message": "Store retrieved successfully",

    "id": "clz9x8w7v6u5t4s3r2q1",
    "name": "Minha Loja",
    "domains": ["loja.exemplo.com.br"],

```

<Tip>
  Use este endpoint para verificar se o webhook está configurado corretamente ou para confirmar quais domínios estão autorizados no seu projeto.
</Tip>

## Erros comuns

| HTTP | Descrição |
|------|-----------|
| `401` | Token de autenticação inválido ou ausente |
| `404` | Projeto não encontrado para a chave fornecida |
