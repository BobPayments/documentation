# Criar QR Code PIX

Cria uma nova transação PIX e retorna o código copia-e-cola para pagamento.

Se uma transação idêntica (mesmo cliente, valor e produto) já estiver pendente, a transação existente será retornada em vez de criar uma nova.

## Autenticação

```
Authorization: Bearer sk_test_xxx
```

## Response (201)

```json
{
  "success": true,
  "message": "Transaction created successfully",
  "data": {
    "id": "clx7a8b9c0d1e2f3g4h5",
    "pixCode": "00020126580014br.gov.bcb.pix...",
    "expirationDate": "2026-01-17T10:00:00.000Z",
    "status": "waiting_payment"
  }
}
```

<ResponseField name="data.id" type="string">
  ID da transação. Use para consultar status e simular pagamento.
</ResponseField>
<ResponseField name="data.pixCode" type="string">
  Código PIX copia-e-cola. Exiba ao cliente ou converta em QR Code.
</ResponseField>
<ResponseField name="data.expirationDate" type="string">
  Data/hora de expiração do PIX (ISO 8601).
</ResponseField>
<ResponseField name="data.status" type="string">
  Status inicial: `waiting_payment`.
</ResponseField>

## Erros

| HTTP | Código | Descrição |
|------|--------|-----------|
| 400 | `ERR_VALIDATION_001` | Body inválido |
| 401 | `ERR_AUTH_009` | Token inválido |
| 403 | `ERR_UNAUTHORIZED_DOMAIN` | Domínio de origem não registrado |
| 403 | `FRAUD_DETECTED` | Limite de transações excedido para este cliente |

| 429 | `ERR_RATE_LIMIT_001` | Muitas requisições |
