# Ciclo de vida da transação

Toda transação PIX passa por um ciclo de vida com status bem definidos.

## Status

| Status | Descrição | Final? |
|--------|-----------|--------|
| `waiting_payment` | PIX gerado, aguardando pagamento | Não |
| `processing` | Pagamento em processamento | Não |
| `paid` | Pagamento confirmado | Sim |
| `expired` | PIX expirou sem pagamento | Sim |
| `cancelled` | Transação cancelada | Sim |
| `refunded` | Pagamento estornado | Sim |
| `failed` | Falha no processamento do pagamento | Sim |

## Fluxo

```
                    ┌─── processing ──┬─── paid
                    │                 │
waiting_payment ────┼─── expired      └─── failed
                    │
                    └─── cancelled

paid ───────────────────── refunded
```

## Regras

- Toda transação inicia com status `waiting_payment`
- O PIX tem tempo de expiração configurável (padrão: 30 minutos)
- Quando o PIX expira, o status muda para `expired` automaticamente
- Transações em status final (`paid`, `expired`, `cancelled`, `refunded`, `failed`) não podem ser alteradas
- Ao consultar uma transação expirada, o status é atualizado automaticamente

## Webhooks

Cada mudança de status gera um evento de webhook:

| Evento | Quando |
|--------|--------|
| `transaction_created` | Transação criada |
| `transaction_paid` | Pagamento confirmado |
| `transaction_expired` | PIX expirou |
| `transaction_cancelled` | Transação cancelada |
| `transaction_refunded` | Pagamento estornado |
| `transaction_failed` | Falha no processamento do pagamento |
