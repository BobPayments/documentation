# Checar Status

Retorna os detalhes completos de uma transação, incluindo status de pagamento e dados do cliente.

Se a transação estiver expirada, o status será atualizado automaticamente para `expired`.

## Autenticação

```
Authorization: Bearer sk_test_xxx
```

## Path Parameters

<ParamField path="id" type="string" required>
  ID da transação (formato CUID).
</ParamField>

<ResponseField name="data.status" type="string">
  Status atual da transação. Valores possíveis: `waiting_payment`, `processing`, `paid`, `expired`, `cancelled`, `refunded`, `failed`.
</ResponseField>
<ResponseField name="data.paidAt" type="string | null">
  Data/hora do pagamento (ISO 8601). `null` se não foi pago.
</ResponseField>

## Erros

| HTTP | Código | Descrição |
|------|--------|-----------|
| 401 | `ERR_AUTH_009` | Token inválido |
| 403 | `ERR_TRANSACTION_003` | Transação pertence a outro projeto |
| 404 | `ERR_TRANSACTION_001` | Transação não encontrada |
